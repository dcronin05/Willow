# Understanding LDtk Integration in Willow

This project uses **LDtk** for visual level design, but we intentionally do **not** use a bloated third-party library to load it into the engine! Instead, we wrote a 40-line custom JSON parser in `World.lua` that natively reads exactly what we need.

This keeps the engine extremely fast, highly educational, and perfectly tailored to our specific use case.

## The Core Files
1. `source/levels/world.ldtk`: The actual JSON file generated by the LDtk Desktop app.
2. `source/scripts/World.lua`: Where we decode the JSON file and dynamically construct the world.

---

## Step 1: Loading the Data (`World.lua`)
At the very top of `World:init()`, we use Playdate's native JSON decoder to read the file into a Lua table, and then find the specific room data:
```lua
local ldtkData = json.decodeFile("levels/world.ldtk")
```

## Step 2: Building the Graphics and Collisions
We iterate over the `layerInstances` in the JSON. When we find the `Collisions` layer (an IntGrid layer), we read its `intGridCsv` array.

LDtk IntGrid values are extremely simple: `0` means empty sky, `1` means a solid Wall. 

We simply loop over the CSV, map the LDtk `1` to our own tileset's Wall Tile ID, and tell Playdate to generate solid physics collisions for everything else!

## Step 3: Spawning the Entities
We placed the Player and Signs dynamically in the LDtk visual editor instead of hardcoding their X/Y coordinates in Lua. 

When our parser hits the `Entities` layer, it checks the string `__identifier` (which matches what we named it in LDtk), extracts its `px` coordinates, and instantiates our custom Lua classes!

```lua
if entity.__identifier == "Sign" then
    -- LDtk allows attaching custom fields to entities!
    -- We loop through fieldInstances to extract the "text" field:
    local text = "..."
    for _, field in ipairs(entity.fieldInstances) do
        if field.__identifier == "text" then
            text = field.__value
        end
    end
    Sign(entity.px[1], entity.px[2], text)
end
```

## How to add a new Entity type (e.g. an Item)
1. Open `world.ldtk` in the LDtk app.
2. Create a new Entity Definition (e.g., `Sword`).
3. Place the Sword in your room using the Entities layer.
4. In `World.lua`, add a new `elseif entity.__identifier == "Sword" then` condition to instantiate your custom `Sword.lua` class!
