# Understanding LDtk Integration in Willow

This project uses **LDtk** for level design and the **PlaydateLDtkImporter** (by Nic Magnier) to parse the JSON files into the Playdate engine. This document explains exactly how it's wired up so you don't forget how the engine functions under the hood.

## The Core Files
1. `source/levels/world.ldtk`: The actual JSON file generated by the LDtk Desktop app.
2. `source/scripts/LDtk.lua`: The third-party library that parses the JSON into Lua tables.
3. `source/main.lua`: Where we tell the library to load the JSON file.
4. `source/scripts/World.lua`: Where we dynamically construct the world using the parsed data.

---

## Step 1: Loading the Data (`main.lua`)
Before doing anything, we load the JSON file into memory:
```lua
import "scripts/LDtk"
LDtk.load("levels/world.ldtk")

-- Then we instantiate our World class, passing in the specific room we want to build
local world = World("Room_1")
```

## Step 2: Building the Graphics and Collisions (`World.lua`)
Inside `World:init(levelName)`, we first tell the library to activate the specific room:
```lua
LDtk.load_level(levelName)
```

Then we iterate over the layers we built in the LDtk app. When we find the `Collisions` layer (an IntGrid layer), we do three things:
1. Generate the visual tilemap (`LDtk.create_tilemap`).
2. Attach it to a Sprite so it draws to the screen.
3. Provide an array of "empty" tile IDs (IDs other than `1`) to `addWallSprites()`. The engine automatically creates solid physics colliders for all the `1` (Wall) tiles!

## Step 3: Spawning the Entities (`World.lua`)
We placed the Player and Signs dynamically in the LDtk visual editor instead of hardcoding their X/Y coordinates in Lua. 

We iterate over the `Entities` layer. For every entity found, we check its string `name` (which matches what we named it in LDtk), extract its position, and instantiate our custom Lua classes!

```lua
for index, entity in ipairs(LDtk.get_entities(levelName)) do
    if entity.name == "Player" then
        _G.player = Player(entity.position.x, entity.position.y)
        
    elseif entity.name == "Sign" then
        -- LDtk allows attaching custom fields to entities!
        -- We extract the "text" field that you typed in the editor:
        local text = entity.fields.text or "..."
        Sign(entity.position.x, entity.position.y, text)
    end
end
```

## How to add a new Entity type (e.g. an Item)
1. Open `world.ldtk` in the LDtk app.
2. Create a new Entity Definition (e.g., `Sword`).
3. Place the Sword in your room using the Entities layer.
4. In `World.lua`, add a new `elseif entity.name == "Sword" then` condition to instantiate your custom `Sword.lua` class!
